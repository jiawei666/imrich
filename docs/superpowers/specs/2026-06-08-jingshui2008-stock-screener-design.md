# i'mRich— 选股器设计文档（v1）

- 日期：2026-06-08
- 数据源：知乎用户 `jingshui2008`（静水2008）124 条回答（`zhihu-jingshui2008/all_answers_combined.md`）
- 本文档原则：**一切以原作者思想为准**。每个选股信号都可追溯到作者原话；作者未给出精确数字的阈值，明确标注为"可调默认值"。

---

## 0. 定位

把作者"亲自看财报、找超级成长 α"的功夫**前移成一个候选漏斗**。它**不替用户做决策**（作者明确说"别全靠 AI，要亲自看财报，AI 打打辅助还行"），而是把全市场几千只 A 股筛成几十只值得深挖的候选，并把命中的财报数据和研报原文摆到用户面前，供人工复核。

非目标（v1 明确不做）：定时推送、回测、持仓/止盈止损管理、港美股、多用户登录、自动下单。

## 1. 作者投资体系的忠实提炼

### 1.1 主纲领（作者 ~90% 回答反复强调）

> **"找到当下时代最景气行业中、核心受益且市场认可的 α，在它回调时分仓分批介入，对了死拿（持盈），错了砍掉（止损）。"**

### 1.2 "好股票"的定义（第35条）

1. **业绩好**：业绩不断刷新历史新高、不断超预期、持续稳定上涨。
2. **市场认**：股价不断刷新历史新高、持续稳定上涨。

### 1.3 抓大趋势的系统三步（第37条）

1. 哪个行业景气度最高、行业内公司业绩**集体在财报期大超预期**、且**行业指数已创历史新高**；
2. 该行业里哪个标的**业绩最好、最受市场认可、涨幅最大**（= α）；
3. 回调时分仓分批介入。

### 1.4 趋势识别（第41条）

> "股价创历史新高。" —— 真传一句话。

### 1.5 财报季关键词法（第3条，最具体的落地方法）

- 时间：每年 **1 / 4 / 7 / 10 月** 财报季。
- 动作：逐个看业绩大增公司的财报、招股说明书。
- 关键词（财报 / 交流 / 调研 / 研报中出现）：
  **产品供不应求、行业高景气度上行、市场超预期拓展、新品上市持续超预期、产品价格中枢持续上涨、供给偏紧、需求旺盛**。
- 判据：**业绩大增 + 命中关键词** → 重点关注。

### 1.6 反向避雷 — 垃圾股特征（第12 / 16 / 26条反复出现）

1. 业绩增速、增量持续下滑；
2. 股价不停创新低、创历史新低（散户最爱的"便宜低位股"）；
3. 产品 / 服务竞争力持续变差。
4. 风控教训（第31条爆仓案例）：**不加杠杆、不单吊（必须分仓）**。

### 1.7 持有/卖出（体系一部分，v1 选股器不实现，仅记录）

- 持盈止损：截断亏损，让利润奔跑。
- 卖出时机（第13 / 27 / 35条）：拿到**公司基本面业绩不行了、涨不动了**再卖，而非按获利幅度止盈。
- 超级成长股通常经历 2 次 30–40% 的中期调整（第17 / 26条），要拿得住。
- 介入：回调时分仓分批，设止损位（第49条）。

### 1.8 主线 vs 特例（关键判断，影响策略定位）

- 作者**主体思想坚决反对买低位、反对预判反转**（第12 / 26 / 42 / 61条措辞极重："恐高是穷人思维""挖掘底部是弯路"）。
- **策略1（创新高·强势版）= 作者绝对主线（~90% 观点）。**
- **策略2（低位错杀蓝筹）= 作者明说的"一个特例方向"**（用户提供的第二篇原文，作者自称"这只是一个方向，大家都在忽视的"），仅在"AI 吸血效应"特定环境下成立，且作者反复警告：**只捞被错杀的绩优股，绝不碰低位景气下行的真垃圾**。
- 产品上必须**显著标注这个主次关系**，避免用户用策略2时滑向作者最反对的"抄底垃圾股"。

## 2. 总体架构

```
数据层 (akshare + 东财研报爬虫)
        ↓  灌入
本地缓存 DB (SQLite)
        ↓  读取
信号层 (原始数据 → 一个个"信号"，纯函数)
        ↓
策略引擎 (信号 → 按 preset 硬过滤/加权打分/避雷) → 候选清单
        ↓
FastAPI (后端 API)  ←→  React (本地交互面板)
```

**核心设计：数据更新与查询解耦。** akshare 与爬虫都慢，故用"数据更新"任务把数据灌入本地 SQLite（财报季手动触发或定时脚本），面板查询全部走本地缓存，保证交互秒级响应。面板顶部显示"数据更新于 X"。

---

## 3. 数据层

### 3.1 akshare 拉取
- A 股股票列表与基本信息（含上市日期、是否 ST）。
- 财报：单季 / 累计 归母净利润、营收、同比、环比、毛利率。
- 业绩预告 / 业绩快报。
- 日线行情（用于计算创新高、回撤、区间涨幅）。
- 申万行业分类 + 申万行业指数行情（用于行业指数创新高、行业景气方向）。

### 3.2 东财研报爬虫
- 抓取个股研报标题 + 摘要 +（可得时）全文 / 评级，存本地。
- 用于：关键词匹配（信号）+ 下钻阅读原文。
- 工程要求：**限速、本地缓存、断点续爬**，避免被封。

### 3.3 缓存
- SQLite，按"数据类型 + 更新日期"组织。
- 记录每类数据的更新时间，前端展示。

## 4. 信号库（可复用底层信号，两个策略共享）

每个信号是**纯函数**：输入财报 / 行情 / 研报 → 输出信号值（布尔或分值）。便于单元测试。

| 信号 | 作者依据（条目） | 量化定义 | 默认阈值 | 数字来源 |
|---|---|---|---|---|
| 业绩大增 | 第3 / 37 | 单季归母净利润同比增长 | 同比 > 50% | ⚠️可调默认值 |
| 业绩超预期 | 第3 / 28 / 37 | 有业绩预告/快报，实际增速 > 历史中枢（或超一致预期） | — | ⚠️可调默认值 |
| 业绩创新高 | 第35 | 单季 / TTM 净利润 = 历史新高 | — | ✅作者明说 |
| 股价创历史新高 | 第35 / 41 | 收盘价 = 历史新高 / 近一年新高 | 距一年高 < 5% | ✅作者明说 |
| 研报关键词命中 | 第3 | 近 N 天研报命中关键词清单（见 1.5 + 扩产），命中数越多分越高 | N = 90 天 | ✅关键词为原文 |
| 板块效应 | 第28 / 37 | 同申万二级行业内"业绩大增 + 关键词命中"标的数 ≥ 阈值 | ≥ 3 只 | ⚠️可调默认值 |
| 行业指数创新高 | 第37 | 申万行业指数 = 历史 / 阶段新高 | — | ✅作者明说 |
| α地位 | 第37 | 行业内按区间涨幅 / 市值 / 业绩增速排序，取头部 | Top N | ✅作者明说 |
| 低位错杀（策略2） | 第二篇 / 第2 | 距一年高点回撤大，但净利润同比仍高增 | 回撤 > 35% 且净利润同比 > 0 | ⚠️可调默认值 |
| 🚫业绩持续下滑 | 第12 / 16 | 净利润同比连续 ≥ 2 季下滑 | 2 季 | ✅作者明说方向 |
| 🚫股价创历史新低 | 第12 / 16 / 26 | 收盘价 = 历史新低 | — | ✅作者明说 |
| 🚫行业景气下行（策略2） | 第二篇 | 行业整体净利润增速为负 / 下行 | — | ✅作者明说方向 |

## 5. 策略引擎 + 两个策略模板（preset）

每个 preset = **硬过滤（必须满足）+ 加权打分（排序）+ 避雷（一票否决）**。

### 策略1 · 创新高超级成长（主线）
- **硬过滤**：业绩大增 ✓ + 股价创新高（或强势）✓ + 研报关键词命中 ✓
- **加分**：业绩超预期、业绩创新高、板块效应、行业指数创新高、α地位
- **避雷**：业绩持续下滑、股价创历史新低、ST/*ST、上市 < 1 年

### 策略2 · 低位错杀蓝筹（特例 · 带警告标）
- **硬过滤**：业绩好（净利润高增且未下滑）✓ + 低位错杀 ✓ + 行业景气（非下行）✓ + 关键词命中 ✓
- **加分**：回撤幅度大但业绩稳、扩产关键词
- **避雷（作者"千万别碰"）**：① 高位跟随概念暴拉的（创新高但无业绩支撑）② 低位且行业景气下行的
- **定位说明（仅文档记录，不做 UI 横幅）**：此为作者特例策略，仅特定环境适用；作者主体思想反对买低位，使用时需人工复核行业景气方向，避免滑向抄底垃圾股。

## 6. 后端 API（FastAPI）

- `POST /refresh` — 触发数据更新（后台任务，返回进度 / 状态）。
- `GET /presets` — 返回可用策略模板及各自参数 schema（用于前端动态生成滑块）。
- `GET /screen?preset=<id>&params=<...>` — 返回候选清单：每只股票的总分、命中了哪些信号、各分项得分、命中的关键词。
- `GET /stock/{code}` — 个股详情：财报增速序列、行情序列（含创新高/回撤标注）、命中的研报原文列表。
- `GET /meta` — 各类数据的更新时间。

---

## 7. 前端面板（React）

- **筛选页**：选 preset；按 `presets` schema 动态生成参数滑块；候选表（按总分排序，每行展示命中信号徽章 + 关键词）。（具体布局由设计稿决定，本文档不预设布局与配色。）
- **个股下钻页**：财报增速折线图；股价图（标注创新高点 / 回撤幅度）；命中研报原文卡片（高亮关键词）。
- 顶部全局显示"数据更新于 X"。

---

## 8. 默认股票池

A 股全市场，**剔除**：ST / *ST、上市不满 1 年、北交所（流动性差）。可在面板调整。

---

## 9. 测试策略

- **信号层**：纯函数，用固定样本数据（构造的财报 / 行情 / 研报）做单元测试，覆盖阈值边界。
- **数据层**：akshare 与爬虫做**离线 mock**，测试不依赖实时网络。
- **策略引擎**：用已知信号组合验证硬过滤 / 打分 / 避雷逻辑正确。

---

## 10. 可扩展性

架构 = "信号库 + 打分引擎 + 策略模板"。作者未来若有新观点，只需新增一个 preset 配置（复用现有信号），或新增一个信号纯函数，无需重写。这正是把作者多篇分散观点持续沉淀的容器。

---

## 11. 风险与免责

- 本系统是研究辅助工具，不构成投资建议；最终决策需人工看财报、跟产业（作者反复强调）。
- 研报关键词为文本匹配，存在误命中；需在下钻页人工阅读原文确认语境。
- "行业景气方向""业绩超预期"为代理指标，不能完全自动判断，需人工复核。
- 阈值类参数无"正确值"，是可调起点。
